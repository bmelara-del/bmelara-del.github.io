Project Overview

Project Name

Professional Portfolio Expansion for Sebastian Ala Torre

This project is an expanded professional portfolio website designed for my client, Sebastian Ala Torre. The purpose of the site is to better promote his professional background, technical skills, work experience, and projects in a way that is clear and appealing to recruiters, employers, and other professional visitors.

Project Overview

This web application will extend my client’s existing portfolio website into a more complete professional platform. While his current portfolio already introduces him, this expanded version will give visitors a deeper view of his experience, technical projects, skills, resume, and contact information. The website is meant to improve the client’s professional visibility and make it easier for recruiters and other visitors to evaluate his background.

The primary users of this website will be job recruiters, hiring managers, internship coordinators, professional connections, and other visitors who want to learn more about the client’s qualifications and work. Secondary users may include classmates, professors, collaborators, and networking contacts.

The website content will include a landing page, an experience page, a projects page, a skills page, a resume page, and a contact page. These pages will work together to present the client’s professional story in a structured and user-friendly way.

Dynamic Functionality on the Website

1. Interactive Project Gallery

Page: Projects

The Projects page will include an interactive gallery or grid of project cards. Users will be able to click on a project to reveal more information, screenshots, technologies used, and links to repositories or live demos.

Purpose: This helps users explore the client’s work in a more engaging and organized way.

Planned Technology: JavaScript for filtering and displaying project details.

Example URL: https://dribbble.com/

2. Expand/Collapse Experience Sections

Page: Experience

The Experience page will use expandable sections so visitors can quickly scan job or leadership entries and then open only the ones they want to read in more detail.

Purpose: This keeps the page clean while still giving access to detailed professional information.

Planned Technology: JavaScript to handle the expand/collapse functionality.

3. Skills Tabs or Accordion

Page: Skills

The Skills page will group technologies into categories such as languages, frameworks, tools, and software. Users will be able to switch between categories through tabs or accordion sections.

Purpose: This makes the page easier to scan and keeps information organized.

Planned Technology: JavaScript for handling tab switching or accordion functionality.

4. Contact Form Validation

Page: Contact

The Contact page will include a form where users can enter their name, email, subject, and message. JavaScript will validate the form before submission to make sure required fields are filled in correctly.

Purpose: This improves usability and ensures visitors provide complete information.

Planned Technology: JavaScript form validation scripts.

5. Loading Projects

Page: Projects or Resume

Javascript functionality may be used to load project details, featured content, or resume-related information. For example, clicking a project card could display more information into a display panel.

Purpose: This creates a smoother and more modern browsing experience.

Planned Technology:JavaScript

Special Notes and Limitations

This project will mainly focus on showing my client’s portfolio and professional information, rather than building a system that stores user data. The contact form will just be a front-end feature to collect basic input, but overall the project will stay within the course limits and won’t include full CRUD functionality.
